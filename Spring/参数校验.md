# Spring 参数校验 -- 基于Java Bean Validation

## 1. 自定义校验注解
```java
// 指定校验器
@Constraint(validatedBy = {CustomConstraintValidator.class})
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.TYPE, ElementType.FIELD, ElementType.METHOD, ElementType.TYPE_USE, ElementType.PARAMETER})
// optional
@Repeatable(CustomConstraint.List.class)
public @interface CustomConstraint {
    // 必选，错误信息，默认值通常为本注解的message属性的全限定名，以便于国际化
    // e.g. "{packegename.CustomConstraint.messgae}"
    // message 实参中，{}引用注解变量，${}使用表达式，所以 { } $ \ 都需要转义
    String message() default "";
    // 必选，与分组校验、校验顺序有个
    Class<?>[] groups() default {};
    // 必选，荷载 
    // todo 
    Class<? extends Payload>[] payload() default {};
    // 可选，校验对象歧义时必选，指明校验返回值（RETURN_VALUE）或参数（PARAMETERS）
    // 默认值通常为IMPLICIT：
    //     非方法或构造器时校验域；无参方法或无参构造器时校验返回值；无返回值方法时校验参数；
    //     有参有返回值方法或有参构造器不校验。
    // 校验对象不适用下抛出ConstraintDeclarationException
    ConstraintTarget validationAppliesTo() default ConstraintTarget.IMPLICIT;

    // optional
    @Documented
    @Retention(RetentionPolicy.RUNTIME)
    @Target({ElementType.FIELD, ElementType.METHOD})
    @interface List {
        CustomConstraint[] value();
    }

    // other parameter ...
}
```

## 2. 自定义校验器
实现ConstraintValidator接口
```java
// T为被校验类型
// 该类并未被实例化为单例，但可以注入Spring Bean
public class CustomConstraintValidator implements ConstraintValidator<CustomConstraint, T> {
    @Override
    public boolean isValid(T value, ConstraintValidatorContext context) {
        // value 需要校验的值，个人认为value应该为final，不过这个value只是一个copy，不同的校验使用不同的copy
        // ConstraintValidatorContext 通常用于修改提示信息

        // 获取提示信息，通常用于判断用户是否自定义了提示信息或国际化提示信息
        context.getDefaultConstraintMessageTemplate();
        // 禁用原提示信息，包括用户自定义
        context.disableDefaultConstraintViolation();
        // 自定义提示信息，若未禁用原提示信息，则会进行多次校验，
        context.buildConstraintViolationWithTemplate("NEW_MESSAGE").addConstraintViolation();

        // ...
    }

    // 该方法每次执行isValid()之前都会被调用
    @Override
    public void initialize(CustomConstraint constraintAnnotation) {
        // constraintAnnotation可以获取注解的属性值
        
        // ...
    }
}
```

## 3. 使用校验 
* @Valid 为Java Bean Validation的注解，用于启动实体类校验
* @Validated 为Spring的注解，通常用在类级别，启用方法参数的校验。 <br/>
（该注解可以用在方法上，但依然必需在该方法的类上使用该注解）

Spring中参数校验通常的做法
```java
@Controller
// 类级别开启校验
@Validated
public TestController {
    // 1. 直接对参数校验
    // 2. 用@Validated或@Valid对实体参数内部属性校验
    //    通常使用@Validated，因为其可以指定校验组
    @RequestMapping
    public void API(@NotEmpty String str, @Validated ENTITY entity) {}
}
```
参数校验可能会抛出不同的异常 <br/>
BindException、MethodArgumentNotValidException、 ConstraintViolationException等 <br/>
全局异常处理需要考虑多种情况

## 4. 分组校验
示例
```java
public interface group0 {}
public interface group1 {}
public interface group2 {}

// str0隐式属于javax.validation.groups.Default组
@Min(3)
private Integer str0;
@Min(value = 5, groups = group1.class)
private Integer str1;
@Min(value = 6, groups = group2.class)
private Integer str2;
@Min(value = 8, groups = {group1.class, group2.class})
private Integer str3;
```
通过@Validated指定校验组，如果未指定，则隐式指定Default组 <br/>
若已指定校验组，且还需要校验默认组，则要显示指定Default组 <br/>
```java
// 校验str0
@Validated
// 校验str1、str3
@Validated(group1.class)
// 校验str1、str2、str3 
@Validated({group1.class, group2.class})
// 校验str0、str1、str2、str3 
@Validated({Default.class, group1.class, group2.class})
// 不校验任何属性，通常直接把@Validated去掉
@Validated(group0.class)
```
