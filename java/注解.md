# Annotation

定义：
```java
public @interface CustomAnnotation {
    // value 为默认参数
    // 参数后可加 default 默认值 
    // 参数类型：基本数据类型、String、Class、Enum、Annotation及其数组
}
```

元注解：

## 1. @Documented
是否包含在javadoc中

## 2. @Retention
注解的存在时间，值为RetentionPolicy
* SOURCE -- 源码，不会存在.class文件中
* CLASS -- 类，即字节码，存在.class文件，但运行时不起作用
* RUNTIME -- 运行时，即机器码，长期存在

## 3. @Target
可注解的对象，值为ElementType数组
* TYPE -- 类
* FIELD -- 属性
* METHOD -- 方法
* PARAMETER -- 方法的参数
* CONSTRUCTOR -- 构造器
* LOCAL_VARIABLE -- 方法中局部变量
* ANNOTATION_TYPE -- 注解
* PACKAGE -- 包
* TYPE_PARAMETER -- 泛型形参
* TYPE_USE -- 泛型形参或泛型实参

由于javac通常不会保留局部变量的注解，LOCAL_VARIABLE通常用在RetentionPolicy.SOURCE。 <br/>
个人认为：由于编译器优化，局部变量可能会被去除，最主要的是反射无法获取局部变量，局部变量注解在运行时无意义。 <br/>
TYPE_PARAMETER：e.g. <@CustomAnnotation T> <br/>
TYPE_USE：e.g. <@CustomAnnotation T extends @CustomAnnotation List> 或 List<@CustomAnnotation String> <br/>

## 4. @Inherited 
可继承，即子类型无需注解也能够拥有父类型的注解
| 注解类型 | 继承情况 |
|---|---|
| TYPE | 类继承类有效，类实现接口、接口继承接口均无效 |
| FIELD | 无论继承或实现，public但未被重写的属性有效，重写后无效 |
| METHOD | 无论继承或实现，public但未被重写的方法有效，重写后无效 |

## 5. @Repeatable
可重复注解，必需提供复数注解的注解，且复数注解的@Target必需为该注解的子集
```java
@Repeatable(CustomAnnotations.class)
public @interface CustomAnnotation {}

public @interface CustomAnnotations {
    CustomAnnotation[] value();
}

// 使用方法
@CustomAnnotation
@CustomAnnotation
// 或
@CustomAnnotations({@CustomAnnotation, @CustomAnnotation})
// 或复合使用以上两种情况

// 内嵌方式，Java Bean Validation规范推荐
@Repeatable(CustomAnnotations.List.class)
public @interface CustomAnnotation {
    @interface List {
        CustomAnnotation[] value();
    }
}
@CustomAnnotation.List({@CustomAnnotation, @CustomAnnotation})
```
